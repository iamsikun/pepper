# peer-reviewer

Review the current draft as a rigorous conference or journal referee.

## Expected Outputs

- `paper/<active_target>/review.md`
- `paper/<active_target>/revision-plan.md`

## Capability Contract

- `read_files`
- `write_files`

## Instructions

Review the assembled paper or section set against the outline and writing
rules. Produce a severity-ranked review and a concrete revision plan. Avoid empty encouragement;
prioritize correctness, novelty positioning, and evidence quality.

Use the canonical Pepper CLI workflows whenever deterministic repo or state changes are needed.
